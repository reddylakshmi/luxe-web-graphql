Luxe GraphQL API

Overview

The Luxe GraphQL API powers the Luxe e-commerce storefront.

All amounts are expressed in cents (USD) — e.g. 2495 = .95.

Environments

Environment URL
Local dev http://localhost:4000/graphql
Staging https://graphql-api.staging.luxe.com/graphql
Production https://graphql-api.luxe.com/graphql

Data source

  • Mock mode (default): resolvers read from data/*.json files. No external dependencies.
  • Production mode: set JAVA_API_URL in .env to proxy all operations to the Java REST backend. No schema or query changes required.

Authentication

The login and register mutations simulate Okta's Authorization Server in mock mode. In production they delegate to the Java auth service which validates Okta JWTs. Pass the returned token as Authorization: Bearer <token>.

Queries

cart

Description

Current user's cart (single shared cart in mock mode)

Response

Returns a Cart!

Example

Query
query cart {
  cart {
    id
    currency
    items {
      ...CartLineItemFragment
    }
    promotions {
      ...CartPromotionFragment
    }
    totals {
      ...CartTotalsFragment
    }
    updatedAt
  }
}
Response
{
  "data": {
    "cart": {
      "id": 4,
      "currency": "abc123",
      "items": [CartLineItem],
      "promotions": [CartPromotion],
      "totals": CartTotals,
      "updatedAt": "abc123"
    }
  }
}

Types

ID

Description

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.

Example
4
